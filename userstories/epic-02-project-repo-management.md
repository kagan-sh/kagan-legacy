# Epic 2: Project and Repository Management

| ID     | Story                                                                                                                                                                     |
| ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| US-010 | As a user, I want to create a project with optional repo path so I can start from either an empty project or existing repo.                                               |
| US-011 | As a user, I want folder-open flow to resolve existing project by repo path or create new project if missing.                                                             |
| US-012 | As a user, I want a Welcome screen with recent projects, task summary, and quick-open shortcuts.                                                                          |
| US-013 | As a user, I want to switch project via keybinding/command palette/system command so navigation is fast.                                                                  |
| US-014 | As a user, I want repo picker and active-repo switching so one project can span many repos.                                                                               |
| US-015 | As a user, I want active repo switch to update runtime context with changes broadcast to all connected clients so every instance reflects the current state in real time. |
| US-016 | As a user, I want first-time repo branch setup from checked-out branch so default base branch is initialized correctly.                                                   |
| US-017 | As an MCP user, I want project operations (list, open, create, add_repo, find_by_repo_path) for automation and scripted workflows.                                        |
