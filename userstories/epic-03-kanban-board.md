# Epic 3: Kanban Board and Task Authoring

| ID     | Story                                                                                                                                        |
| ------ | -------------------------------------------------------------------------------------------------------------------------------------------- |
| US-018 | As a user, I want to create PAIR or AUTO tasks from board actions so I can choose execution mode at creation time.                           |
| US-019 | As a user, I want task fields for title, description, acceptance criteria, priority, type, agent backend, terminal backend, and base branch. |
| US-020 | As a user, I want task mention autocomplete in description input so cross-task context can be linked quickly.                                |
| US-021 | As a user, I want inline view/edit task details modal and full-screen description editor so short and long edits are both ergonomic.         |
| US-022 | As a user, I want duplicate-task workflow so I can reuse structure of prior tasks.                                                           |
| US-023 | As a user, I want delete-task flow to safely stop agent/session and clean workspace so deletion is coordinated.                              |
| US-024 | As a user, I want task search/filter on board so I can quickly locate work items.                                                            |
| US-025 | As a user, I want to copy short task ID from board/details for sharing in chat/PRs.                                                          |
| US-026 | As a user, I want peek overlay with runtime/session/scratchpad snapshot so I can inspect context without opening modals.                     |
| US-027 | As an MCP user, I want task_get/list/context/logs/search coverage so external tools can read task state deeply.                              |
