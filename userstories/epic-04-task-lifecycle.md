# Epic 4: Task Lifecycle Rules and Governance

| ID | Story |
|----|-------|
| US-028 | As a user, I want lifecycle states BACKLOG -> IN_PROGRESS -> REVIEW -> DONE consistently enforced. |
| US-029 | As a user, I want direct DONE mutation blocked from generic move/patch/update flows so completion always goes through review/merge/no-change close. |
| US-030 | As a user, I want task type (AUTO vs PAIR) separated from status so invalid status values are rejected with guidance. |
| US-031 | As a user, I want task-type transitions to handle side effects (stop AUTO, kill PAIR session) so mode changes are safe. |
| US-032 | As a user, I want branch override per task so review/merge targets are explicit. |
| US-033 | As a user, I want scratchpad append/read operations so AI and humans can keep incremental notes. |
