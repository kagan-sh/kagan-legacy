# Epic 6: PAIR Execution and Session Handoff

| ID | Story |
|----|-------|
| US-038 | As a user, I want session creation bound to task-scoped MCP credentials so pair workflows stay isolated and secure. |
| US-039 | As a user, I want backend support for tmux, vscode, and cursor with platform-aware fallback behavior. |
| US-040 | As a user, I want pre-launch instructions modal with "don't show again" preference. |
| US-041 | As a user, I want terminal/backend readiness checks and install guidance before attach. |
| US-042 | As a user, I want session attach/re-attach and existence checks so workflow can resume after interruptions. |
| US-043 | As a user, I want automatic status handling (e.g., BACKLOG to IN_PROGRESS on open; optional prompt to move to REVIEW on completion). |
| US-044 | As an MCP user, I want session_manage(action=open|read|close) so session lifecycle is automatable. |
| US-045 | As a user, I want task/session env vars injected (KAGAN_TASK_ID, worktree path, etc.) so external tools have runtime context. |
