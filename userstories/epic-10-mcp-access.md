# Epic 10: MCP Access and Monitoring

The MCP server exposes the same operations available in the TUI orchestrator overlay,
structured as machine-consumable tools for AI clients and automation.

| ID | Story |
|----|-------|
| US-094 | As an MCP user, I want read-only mode (--readonly) for safe observation-only sessions. |
| US-095 | As a user, I want capability profiles (viewer, planner, pair_worker, operator, maintainer) to constrain tool access per session. |
| US-096 | As an MCP user, I want long-poll wait (task_wait) with cursor resume for race-safe task monitoring. |
