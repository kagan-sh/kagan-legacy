# Epic 11: CLI Operations and Maintenance

| ID | Story |
|----|-------|
| US-097 | As a user, I want a default `kagan` command that launches TUI while still supporting explicit `kagan tui`. |
| US-099 | As a user, I want `kagan core start/status/stop` to manage daemon lifecycle. |
| US-100 | As a user, I want `kagan doctor` with PASS/WARN/FAIL and concrete fix hints. |
| US-101 | As an MCP user, I want `kagan mcp` with flags for capability, identity, session scope, endpoint override, and diagnostics. |
| US-102 | As a user, I want `kagan update` check/install with install-method detection (uv, pipx, pip) and prerelease support. |
| US-103 | As a user, I want startup update checks with opt-out flag/environment override. |
| US-104 | As a user, I want `kagan reset` to support both full wipe and project-scoped deletion with confirmation gates. |
| US-105 | As a user, I want `kagan list` to show projects, repos, and per-status task counts. |
| US-106 | As a user, I want `kagan tools enhance` for one-shot prompt enhancement (arg or file input). |
